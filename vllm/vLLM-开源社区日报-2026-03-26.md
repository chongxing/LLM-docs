## 📋 重点技术动态

### 1. **MoE 与专家并行 (EP)**
- **#38160** - 新增 `hybrid_ep` 后端，支持混合专家并行策略
- **#38144** - 弹性EP peer-swap恢复的替换rank引导机制 (RFC)
- **#38050** - FlashInfer NVFP4 Cutlass MoE kernel 集成
- **#37880** - 支持通过 `--speculative-config` 为每个草稿模型指定MoE后端

### 2. **ROCm/AMD 支持进展**
- **#38155** - 为 MI355 添加 Qwen3.5 模型的 LM Eval 测试
- **#38146, #38145** - 修复 AITER MLA decode 的 bug (backport到v0.13.0)
- **#38137** - 修复 AITER 状态泄漏问题
- **#38088** - 增加 OpenAPI schema 测试超时时间

### 3. **性能优化**
- **#38139** - CPU-only pooling token IDs 移除冗余设备拷贝，E2E吞吐提升48.9%
- **#37970** - 优化 SM120 CUTLASS blockwise FP8 GEMM

### 4. **模型支持**
- **#38120** - 启用 Cohere-Transcribe 模型支持
- **#38156** - NanoNemotron-VL (RADIO ViT) 批量视频编码
- **#38140** - 修复 Qwen 3.5 GGUF 加载问题

### 5. **V1 引擎**
- **#38157** - 新增 `--health-port` 参数支持带外健康检查进程
- **#38154** - 改进 DCP 错误消息，提示 `VLLM_ATTENTION_BACKEND` 替代方案
- **#38096** - KV Connector 移除 num_cached_tokens 在错误处理中的使用

### 6. **结构化输出与工具调用**
- **#38150** - Mistral Grammar 支持 Grammar Factory
- **#38158** - 修复 n>1 流式工具调用中的共享对象别名问题
- **#38143** - Responses API harmony 路径中应用截断
