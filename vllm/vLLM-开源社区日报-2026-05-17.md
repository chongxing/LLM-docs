## ✅ 今日新增PR

- [PR 42855 - [Bugfix] Fix DSV4 Base model swiglu limit issue in FP8 path](https://github.com/vllm-project/vllm/pull/42855) - @zx3xyy `bug` `nvidia`
  - 💡 技术分析: 量化相关, DeepSeek V4 支持, Bug 修复
  - DeepSeek V4 Pro Base 的 FP8 routed MoE 需要应用模型提供的 SwiGLU clamp limit，否则会产生乱码输出。该PR修复了 FP8 MoE 后端('auto' 和 'triton')的这个问题，修复后 MMLU Pro benchmark 的 macro/micro 分数与 model card 匹配。
- [PR 42854 - [Bugfix] Scope deprecated CLI argument warnings to parser instances](https://github.com/vllm-project/vllm/pull/42854) - @ThomsonTan `bug`
  - 💡 技术分析: Bug 修复, 前端/CLI
  - 修复 `FlexibleArgumentParser` 中 deprecated action 存储在 parser class 上导致跨 subcommand 泄漏警告的问题。例如 `vllm chat --url` 会错误地发出 `--url` 已弃用的警告。
- [PR 42853 - structured_output: skip accept_tokens/validate_tokens on terminated grammar](https://github.com/vllm-project/vllm/pull/42853) - @notandruu `v1`
  - 💡 技术分析: v1 引擎, Spec Decode, Bug 修复
  - 修复 structured output 请求在完成时会被错误标记为 `FINISHED_ERROR` 的问题。当 grammar FSM 到达终止状态后，后续的 `accept_tokens()` 调用返回 False，调度器将其视为 grammar violation。该 PR 在三个调用点添加 `is_terminated()` 保护。
- [PR 42852 - [Bug]: Deprecated CLI argument warning leaks from run-batch --url to chat --url](https://github.com/vllm-project/vllm/issues/42852) - @ThomsonTan `bug`
  - 💡 技术分析: Bug 报告, 前端/CLI
  - 与 #42854 相关的 issue，`vllm chat --url http://127.0.0.1:8000/v1` 会错误触发 deprecation warning。
- [PR 42851 - Refactor: Pass num_labels explicitly to PoolerClassify instead of reading from global config](https://github.com/vllm-project/vllm/pull/42851) - @taneem-ibrahim
  - 💡 技术分析: 代码重构, Pooler 模块
  - `PoolerClassify` 不再通过 `get_current_vllm_config()` 从全局状态读取 `num_labels`，而是直接作为参数接收。这移除了测试中对 `VllmConfig` 上下文的依赖。
- [PR 42850 - [Perf] Eliminate two GPU→CPU syncs in make_kv_sharing_fast_prefill_common_attn_metadata](https://github.com/vllm-project/vllm/pull/42850) - @wojciech-wais `v1`
  - 💡 技术分析: 性能优化, v1 引擎, KV Cache 相关
  - 消除了 `make_kv_sharing_fast_prefill_common_attn_metadata()` 中的两处 GPU→CPU 同步点。将 `num_decode_tokens` 的 `.max().item()` 和 `.sum().item()` 改为在 CPU 上通过 `decode_query_start_loc` 的 cumsum 张量计算，减少同步开销。
- [PR 42849 - [Perf] Add do_not_specialize in fused FP8 RoPE kernel](https://github.com/vllm-project/vllm/pull/42849) - @xyang16 `performance` `v1`
  - 💡 技术分析: 性能优化, v1 引擎, 量化相关, CUDA/Triton 算子优化
  - 在 `_fused_inv_rope_fp8_quant_per_head` kernel 中添加 `do_not_specialize`，防止推理期间因不同 `num_tokens` 导致 Triton kernel 重复编译。在 H200 上 e2e TTFT 提升 9%。
- [PR 42848 - [Core] Update PyTorch to 2.12.0, torchvision to 0.27.0, triton to 3.7.0](https://github.com/vllm-project/vllm/pull/42848) - @atalman `ci/build` `cpu` `nvidia` (draft)
  - 💡 技术分析: CI/Build 相关, PyTorch 生态升级
  - 将 PyTorch 生态升级到 2.12.0/0.27.0/3.7.0，并同步更新 CUDA 13 相关依赖（cudnn, cusparselt, nccl）。

## 🎯 重点关注

### 性能优化
- **#42850** 消除了 KV-sharing fast-prefill 路径中的两处 GPU→CPU 同步，对混合 prefill+decode 场景有明显收益。
- **#42849** 为 fused FP8 RoPE kernel 添加 `do_not_specialize`，避免 Triton 重复编译，H200 上 TTFT 提升 9%。

### 新模型支持
- **#42716** 修复 Qwen3.5-MTP 和 Qwen3-VL 在 TPU 上的权重加载问题，`expert_id` 参数丢失导致 MoE 路由异常。
- **#42855** DeepSeek V4 Base 模型 FP8 路径的 SwiGLU limit 修复，确保量化后输出质量与 model card 一致。

### 调度器与 V1 引擎改进
- **#42853** 修复 V1 引擎 structured output 的 grammar 终止状态处理，避免正常完成的请求被错误标记为 `FINISHED_ERROR`，同时修复了 spec-decode 路径下的相关逻辑。

### ROCm 支持
- **#42807** 清理 ROCm CI 的命令覆盖机制，减少构建问题。
- **#42304** 实验性支持可中断 CUDA graph，对长序列推理场景有潜在收益。

### 量化与 Spec Decode
- **#42855** DeepSeek V4 FP8 MoE 的 SwiGLU clamp 修复，直接影响 FP8 量化精度。
- **#42849** FP8 RoPE kernel 编译优化，减少推理延迟抖动。
- **#42853** 修复 grammar terminated 状态下 spec-decode draft tokens 的验证逻辑。
