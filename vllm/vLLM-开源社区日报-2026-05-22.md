## 💡 社区洞察

### 重点方向

1. **Rust 前端里程碑** — [#40848](https://github.com/vllm-project/vllm/pull/40848) 已合并，标志着 vLLM 正式引入 Rust 前端集成，这是架构层面的重大变革，将提升前端的性能和安全性。

2. **KV Connector 生态扩展** — 今日新增 3 个 KV Connector 相关 PR：
   - NIXL 连接器支持 Pipeline-Parallel PD 分离部署 ([#43366](https://github.com/vllm-project/vllm/pull/43366))
   - NIXL 支持 Mamba/SSM 混合模型的 per-layer HMA 路由 ([#43368](https://github.com/vllm-project/vllm/pull/43368))
   - Mooncake 连接器支持 preemption 后的 finish 处理 ([#43281](https://github.com/vllm-project/vllm/pull/43281))

3. **Spec Decode 性能突破** — [#43349](https://github.com/vllm-project/vllm/pull/43349) 优化 Eagle3 和 DFlash 的贪婪草稿 token 选择，内核性能提升 **37%~81%**。

4. **ROCm 持续投入** — 多个 ROCm 相关 PR：
   - QK-Norm+RoPE 融合，通过 AITER 自动启用 ([#43359](https://github.com/vllm-project/vllm/pull/43359))
   - DeepSeek V4 CSA 多流解码重叠 ([#43365](https://github.com/vllm-project/vllm/pull/43365))
   - AITER top-k/top-p sampler 默认启用 ([#43331](https://github.com/vllm-project/vllm/pull/43331))

5. **量化与 MoE** — NVFP4 支持持续完善：
   - 新增 DeepSeek V4 NVFP4 MoE 支持 ([#42209](https://github.com/vllm-project/vllm/pull/42209))
   - KV cache skip-layers 滑动窗口支持 nvfp4 kv ([#42890](https://github.com/vllm-project/vllm/pull/42890))
   - W4A16 NVFP4 fused MoE + 混合精度分发 ([#42566](https://github.com/vllm-project/vllm/pull/42566))

6. **基础设施改进** — 新增 native ModelExpress 加载格式 ([#43105](https://github.com/vllm-project/vllm/pull/43105))，以及 CPU 分布式超时 CLI 选项 ([#42968](https://github.com/vllm-project/vllm/pull/42968))。
