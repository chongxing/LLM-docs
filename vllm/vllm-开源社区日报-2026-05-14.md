## 🔍 技术趋势与洞察

### 1. V1 引擎持续迭代 (v1 标签 46 个 PR)
- V1 引擎已成为 vLLM 核心发展方向，大量 PR 围绕 V1 的稳定性、性能和功能完善
- ubatch wrapper 支持 tuple model outputs（spec decode + DBO 场景）
- 移除废弃的 MLA prefill 参数，优化 MLA attention 的 `_v_up_proj` bmm

### 2. DeepSeek V4 / GPT-OSS 生态
- DeepSeek V4 Flash 新增 SM12x (Blackwell) 支持
- MTP HC state 处理修复
- cuteDSL generic LL router GEMM 性能优化
- GPT-OSS 工具调用解析在 Harmony 中的兼容处理
- MI300 上融合 MXFP4 SwiGLU fast path

### 3. 量化与压缩 (NVFP4 / W4A16 / Quark)
- Quark NVFP4 checkpoint 加载支持（AMD + NVIDIA）
- W4A16 NVFP4 fused MoE + mixed-precision dispatch
- quantization_config 重构，支持 QuantKey 和格式扩展
- FlashAttention 4 FP8 支持推进中

### 4. PD 分离 (Prefix Caching / KV Connector)
- NIXL EP 安装修复
- Mamba prefix caching mode 支持
- async KV loads 不计入 scheduler request limit 修复
- mooncake-transfer-engine 从源码构建的 Docker 支持

### 5. Speculative Decoding
- 移除 verifier model type check，提升灵活性
- 支持 hybrid attention models 在 extract_hidden_states 中
- Flash rejection sampling (Model Runner V2) 开发中

### 6. 多模态 (MM)
- urllib3 不再用于解析 data URLs（安全/兼容性修复）
- Qwen2-VL ViT CUDA Graph 支持
- OpenVLA 模型支持（已关闭）
